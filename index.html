CatGame

Este código implementa el juego del Gato utilizando la biblioteca tkinter para la interfaz gráfica y contiene la lógica del juego en dos clases: Cat, que maneja las reglas y la lógica interna, y Board, que gestiona la interfaz gráfica y la interacción con el usuario. El juego permite a dos jugadores alternar turnos, hacer clic en las celdas para realizar movimientos, verificar si hay un ganador o empate, y reiniciar el juego cuando es necesario.

En general el juego hace lo siguiente:

  • Inicia el juego en una ventana grafica.
  • Alterna turnos entre los jugadores.
  • Permite hacer clic en las celdas del tablero para jugar.
  • Verifica si hay un ganador o empate.
  • Muestra un mensaje con el resultado.
  • Reinicia el juego cuando termina.

Clase Cat

current_player: Es un string que almacena el jugador actual, ya sea "O" o "X".

board: Es una lista de listas (una matriz de 3x3) que representa el tablero del juego, donde cada celda puede contener "O" o "X".

Métodos:

  • void init():
    Constructor que inicializa el juego. Establece al jugador actual como "O" y crea un tablero vacío (una lista 3x3 llena de strings vacíos).
  • void switch_player():
    Cambia el jugador actual. Si es "O", lo cambia a "X", y viceversa.
  • boolean make_move(int x, int y):
    Intenta hacer un movimiento en la posición (x, y) del tablero. Si la celda está vacía, coloca el símbolo del jugador actual y devuelve True, indicando que el movimiento fue exitoso. Si la celda ya está ocupada, devuelve False.
  • boolean check_winner():
    Verifica si el jugador actual ha ganado. Revisa si hay tres símbolos iguales en una fila, columna o diagonal. Si hay un ganador, devuelve True; de lo contrario, devuelve False.
  • Void reset_game():
    Reinicia el juego, estableciendo el tablero en blanco y asignando el turno al jugador "O".

Clase Board

root:
Es la ventana principal de la aplicación gráfica, que pertenece al tipo Tk (de la biblioteca tkinter).

cat_game:
Es una instancia de la clase Cat, que contiene la lógica del juego.

buttons:
Es una lista de listas (una matriz de 3x3) de botones que representan las celdas del tablero en la interfaz gráfica. Cada botón está asociado a una celda en el tablero y permite a los jugadores hacer clic para realizar un movimiento.

Métodos:

  • Void init(root):
    Constructor que inicializa la interfaz gráfica. Crea la ventana principal del juego y el tablero de botones. Cada botón está asociado a un comando que llama al método handle_click.
  • Void handle_click(int x, int y):
    Maneja el evento de clic en una celda específica (x, y). Si el movimiento es válido (es decir, la celda está vacía), actualiza el texto del botón con el símbolo del jugador actual y verifica si hay un ganador. Si hay un ganador, muestra un mensaje de victoria y reinicia el juego. Si no hay movimientos válidos restantes, declara un empate y también reinicia el juego.
  • Void reset_game():
    Reinicia el juego tanto en la lógica como en la interfaz gráfica. Borra el texto de todos los botones y reinicia el estado del juego llamando al método reset_game de la clase Cat.
